# 正誤判定（judge） Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** ユーザーが実行したSQLの結果が、問題の期待結果と一致するかどうかを判定する `judge()` 純粋関数を実装する。

**Architecture:** `packages/shared` の `Problem` 型に `orderMatters: boolean` を追加してから、`apps/web/src/judge/index.ts` に判定ロジックを実装する2段階構成。判定は「各行の値をJSON.stringify+ソートして正規化文字列化 → `orderMatters` に応じて配列比較 or 多重集合比較」で行う。

**Tech Stack:** TypeScript, Vitest（既存のリポジトリ構成をそのまま利用。新規依存パッケージなし）

## Global Constraints

- 設計書: `docs/superpowers/specs/2026-07-18-judge-design.md`（承認済み）
- `judge(actual: TableResult, problem: Problem): boolean` は純粋関数。SQL実行やDB操作などの副作用を持たない
- 判定基準: 列名・列順は無視し値の集合として比較する。行順は `Problem.orderMatters`（`true`なら厳密な順序比較、`false`なら多重集合として比較）で切り替える
- `Problem.orderMatters: boolean` は必須フィールド（`parseProblem` で型チェックする。既存フィールドと同じ流儀）
- 対象外（本計画では実装しない）: `App.tsx` への組み込み・画面表示、レビュー生成ロジック、XP・レベルシステム、不正解理由の詳細化、問題データの拡充
- テスト: 既存の `vitest@4.1.10` ルート単一設定をそのまま使う（`{apps,packages}/*/**/*.test.ts` グロブで新規テストも拾われる）

```ts
import type { Problem } from "@sql-practice/shared";
import type { TableResult } from "../db/queryResult";

function canonicalRow(values: unknown[]): string {
  return values
    .map((v) => JSON.stringify(v))
    .sort()
    .join(" ");
}

function multisetEqual(a: string[], b: string[]): boolean {
  if (a.length !== b.length) return false;
  const sortedA = [...a].sort();
  const sortedB = [...b].sort();
  return sortedA.every((value, i) => value === sortedB[i]);
}

export function judge(actual: TableResult, problem: Problem): boolean {
  const actualRows = actual.rows.map(canonicalRow);
  const expectedRows = problem.expectedResult.map((row) => canonicalRow(Object.values(row)));

  if (problem.orderMatters) {
    return (
      actualRows.length === expectedRows.length &&
      actualRows.every((row, i) => row === expectedRows[i])
    );
  }

  return multisetEqual(actualRows, expectedRows);
}
```
